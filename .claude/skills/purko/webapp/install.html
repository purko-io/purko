by hand

Manual install

One Helm command. Takes about two minutes on a fresh cluster. All steps below, including verification and the first thing to do once it's running.

Continue reading ↓

Prerequisites
✓
A Kubernetes cluster
minikube, kind, k3s, or any cluster you already have. If you don't have one, minikube is the quickest: minikube start
✓
kubectl
Must be connected to your cluster (kubectl cluster-info should respond).
✓
Helm 3
Version 3.8 or later for OCI chart support. Check with helm version.
Install
helm
helm install purko oci://ghcr.io/purko-io/purko/purko --namespace purko-system --create-namespace

This creates the purko-system namespace and installs the operator and dashboard.

◈

Heads up: container images are being made public. If you hit ImagePullBackOff today, the registry hasn't opened yet — drop us a line and we'll sort it out.

Verify
kubectl
kubectl -n purko-system rollout status deploy/purko-operator

Wait until you see successfully rolled out. Usually under 90 seconds.

Then
next steps
Tell the skill to open Mission Control — it will handle the port-forward automatically:
open mission control
Or forward the dashboard port yourself and open it in a browser:
kubectl
kubectl -n purko-system port-forward deploy/purko-operator 8082:8082

Fresh installs start in auth.mode: open — no login required. Mission Control will show an empty state with an offer to load a starter agent team.
