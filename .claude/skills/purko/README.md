# Purko Skill — Mission Control for Claude Code

A Claude Code skill for watching and managing your [Purko](https://purko.io) AI
agent teams. Say **`/purko`** and it opens a live **Mission Control** webapp
over your cluster — watch workflows and agents in real time, approve human
gates, and (in cluster mode) bootstrap Purko onto a bare cluster. It also
guides agent / workflow / skill authoring and links you to the full dashboard
for per-user namespace management.

## Install

Requires **Python 3** (the skill's helpers are stdlib-only — no `pip` installs).

One line — install or update, from any directory:

```sh
curl -fsSL https://raw.githubusercontent.com/purko-io/purko/main/scripts/install-skill.sh | sh
```

That drops the skill into `~/.claude/skills/purko`, so `/purko` works in every
Claude Code session. Prefer to do it by hand? Clone the repo and copy the dir:

```sh
git clone https://github.com/purko-io/purko.git
cp -R purko/.claude/skills/purko ~/.claude/skills/purko
```

> Working *inside* a clone of `purko-io/purko`? The skill is already available
> as a project skill there — no install needed.

## Use

Run **`/purko`** in Claude Code. On first run it guides you through connecting:

- **Cluster** — you have `kubectl` access to a cluster running Purko (or a bare
  cluster to bootstrap onto).
- **Hosted / SaaS** — connect by workspace URL + token (or a personal access
  token) with no `kubectl` needed.
- **No account yet** — try the offline demo studio, or see how to get access /
  self-host.

Then Mission Control opens at `http://127.0.0.1:8090`, live-updating every ~3s.
Ask the skill things like *"create an agent that writes release notes"* or
*"add the pdf skill to my reviewer"* and it walks you through it.

## Updating

Re-run the one-line installer above. The skill also checks for a newer
published version on `/purko` and shows an **update** chip in Mission Control's
masthead when one is available. See [CHANGELOG.md](./CHANGELOG.md).

## What's inside

| Path | Purpose |
|------|---------|
| `SKILL.md` | the skill definition Claude follows |
| `scripts/` | stdlib-only Python: the local server, config, cluster + token helpers, version check |
| `webapp/` | Mission Control UI, plus the demo / onboarding / install pages |
| `references/` | authoring guides (agents, workflows, MCP) and showcase templates |
| `tests/`, `tests-webapp/` | Python + webapp test suites |

Canonical copy lives at `.claude/skills/purko/`; `.agents/skills/purko/` is a
generated byte-identical mirror (kept in sync by
`scripts/sync-skill-mirror.sh`, enforced in CI).
